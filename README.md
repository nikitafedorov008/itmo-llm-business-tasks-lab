# Персональный ассистент по здоровью

Flutter-приложение для веб-платформы, реализующее персонального ассистента по здоровью с интеграцией ChatGPT API.

## Описание

Приложение предоставляет пользователям возможность общаться с AI-ассистентом по вопросам здоровья, отслеживать свои данные, устанавливать напоминания о приеме лекарств и анализировать симптомы с медицинским дисклеймером.

## Функционал

- **Главная страница**: Описание возможностей и быстрые ссылки на основные функции
- **Профиль**: Форма для ввода персональных данных (имя, возраст, пол, вес, рост) с сохранением в localStorage
- **Чат**: Интерфейс для общения с AI-ассистентом по вопросам здоровья
- **Напоминания**: Простая система напоминаний с localStorage
- **Справка**: Детальная информация о приложении и инструкции по использованию

## Технические требования

- **Платформа**: Flutter web
- **Зависимости**:
  - http: ^1.1.0
  - shared_preferences: ^2.2.2
  - intl: ^0.18.1
  - flutter_dotenv: ^5.1.0
  - provider: ^6.0.5

## Установка и запуск

1. Клонируйте репозиторий
2. Установите зависимости:
   ```bash
   flutter pub get
   ```
3. Создайте файл `.env` в корне проекта и укажите API ключ для ChatGPT:
   ```
   CHATGPT_API_KEY=ваш_api_ключ_здесь
   ```
4. Запустите приложение:
   ```bash
   flutter run -d chrome
   ```

## Архитектура

- `lib/main.dart` - точка входа
- `lib/providers/` - провайдеры состояния (ProfileProvider, ChatProvider)
- `lib/services/` - сервисы (ChatGPT, localStorage, symptom analyzer)
- `lib/screens/` - экраны приложения (Home, Profile, Chat, Help, Reminders)
- `lib/widgets/` - переиспользуемые виджеты (MedicalDisclaimer, CustomAppBar)

## Безопасность и этика

- Все медицинские ответы содержат обязательный дисклеймер: "ВАЖНО: Я не врач. При серьезных симптомах немедленно обратитесь к специалисту."
- Ограничение длины запросов к ChatGPT
- Валидация ввода пользователя
- Защита API ключа через .env файл

## Особенности реализации

- Хранение данных только в localStorage (без внешних БД)
- Адаптивный интерфейс для мобильных и десктопных устройств
- Минималистичный дизайн в медицинской тематике (голубые/зеленые тона)
- Простая форма ввода для чата с кнопкой отправки
- Система кэширования ответов (через localStorage)
- Обработка ошибок API запросов
